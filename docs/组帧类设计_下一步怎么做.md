# 下一步：组帧类怎么做？要不要缓冲区？

> 本文只新增说明文档，**不直接改你工程源码**。  
> 结论先说：**要**。组帧类里应有一个 **`QByteArray` 应用层接收缓冲**，专门缓存从串口来的**原始二进制**，再从中切出完整帧。

---

## 一、为什么必须在组帧类里再建一个缓冲区？

你当前的 `SerialPort` 在 `readyRead` 里：

1. `readAll()` 得到一块 **`chunk`**（长度不固定）；  
2. 立刻 `emit dataReceived(chunk)`。

**`chunk` 往往不是「刚好一整帧」**：可能是半帧、一帧、多帧粘在一起。若不在某处**把多段 `chunk` 拼起来**，就无法稳定地：

- 判断 `0xAA 0xAA` 同步头；  
- 读第 3 字节的 **`PayloadLength = N`**；  
- 再凑齐 **`N` 字节 payload + 1 字节校验**。

因此：**在组帧类里维护 `m_rxBuffer`，每次 `append(chunk)`，再在缓冲上循环切帧**，是标准做法。  
**不要**把这件事塞回 `SerialPort`（否则串口类又管 IO 又管协议，难测、难维护）。

---

## 二、数据流（推荐架构）

```
串口驱动 / QSerialPort
        ↓ readAll()
SerialPort::dataReceived(QByteArray chunk)   ← 仍是二进制，无持久缓冲
        ↓ connect
组帧类::onBytes(chunk)
        ↓ m_rxBuffer.append(chunk)         ← 二进制缓存在这里
        ↓ while 能切出一帧就切
emit frameReady(QByteArray oneFrame)       ← 每帧从第一个 0xAA 起，长度 4+N
        ↓ connect
解析类 / parserData(...)                   ← 只对完整帧做 TLV
```

**十六进制**：只在调试或 UI 里对 `chunk` 或 `oneFrame` 调用 `toHexSpaced`，**不要**把 `m_rxBuffer` 改成十六进制字符串。

---

## 三、组帧类里建议有哪些成员？

| 成员 | 作用 |
|------|------|
| `QByteArray m_rxBuffer` | **缓存收到的原始二进制**，直到凑够至少一帧 |
| （可选）`static constexpr int kMaxRxBytes = 64 * 1024` | 防止异常洪水撑爆内存，超限时清空或保留尾部并打日志 |
| （可选）统计量 | `framesOk`、`checksumFail`、`resyncCount` 便于调试 |

**不需要**在组帧类里放 `QSerialPort`；它只接收 `QByteArray`。

---

## 四、组帧算法（ThinkGear 风格一帧）

一帧布局（与 `docs/脑电串口解析与缓冲设计教学.md` 一致）：

```
[0xAA][0xAA][N][Payload × N][Checksum]
```

- 整帧长度：`frameLen = 4 + N`（`3 + N + 1`）。  
- **`N` 合法范围**：需与设备文档一致；你之前解析里用过 **`N == 0` 或 `N > 170` 视为非法**，可丢弃同步并重新对齐。

**循环逻辑（文字版）：**

1. 在 `m_rxBuffer` 里查找子串 `0xAA 0xAA`。  
2. 若找不到：可保留最后 1 字节（防止 `AA` 被截断），其余丢弃或整缓冲清空（策略二选一，文档里写清）。  
3. 若找到但下标 `i > 0`：删除 `[0, i)` 的垃圾字节。  
4. 若当前长度 `< 3`：等待下次 `onBytes`。  
5. 读 `N = (unsigned char)m_rxBuffer[2]`。若非法：`remove(0,1)` 或从第二个 `AA` 重新搜，避免死锁在误同步上。  
6. 若 `m_rxBuffer.size() < 4 + N`：**帧不完整**，`return`，等更多数据。  
7. 否则：截取 `oneFrame = left(4+N)`，`m_rxBuffer.remove(0, 4+N)`，**`emit frameReady(oneFrame)`**，然后 **继续 `while`** 看是否还有下一帧。

校验和是否验证：应在**组帧类**统一完成（失败则丢弃该帧并计数）。本仓库 `ThinkGearFrameAssembler::checksumOk`：**对全部 `N` 字节 Payload 求和**，再 `expected = (~sum) & 0xFF`，与最后一字节比较。这与思知教程「Payload 四字节的和取反取低 8 位」在小包 `N=4` 时完全一致；**不要**在解析类里再用另一套规则算一遍。

（可选）`setVerifyChecksum(false)` 仅用于对照抓包调试，正式采集应开启校验。

## 五、和 `SerialPort` 怎么连？

在 `MainWindow` 或某个「设备管理」对象里：

```cpp
// 伪代码，仅说明连接关系
auto *serial = new SerialPort(this);
auto *assembler = new ThinkGearFrameAssembler(this); // 类名自定

connect(serial, &SerialPort::dataReceived,
        assembler, &ThinkGearFrameAssembler::onBytes);
```

`ThinkGearFrameAssembler` 需要是 `QObject` 子类，槽函数 `onBytes(const QByteArray &)`。

## 七、验收标准（你可以自测）

1. 不接设备时，`m_rxBuffer` 为空或保持很小。  
2. 接上设备后，`frameReady` 触发频率与设备发包一致；用日志打印每帧长度应合理（例如常见 `N` 为几十字节量级，视固件而定）。  
3. 长时间运行 `m_rxBuffer` 不会无限增长（说明切帧逻辑在消费数据）。  
4. 拔掉设备后可选：`clear()` 缓冲，避免下次插上时旧数据干扰。

## 八、直接回答你的问题

| 问题 | 答案 |
|------|------|
| 组帧类要不要写缓冲区？ | **要**。用 **`QByteArray m_rxBuffer`** 缓存**原始二进制**。 |
| 缓冲放在 `SerialPort` 里行吗？ | **不推荐**。串口层只传 **`chunk`**；组帧与协议状态放在**单独类**更清晰。 |
| 先转十六进制再缓冲？ | **不要**。始终 **二进制 append**，十六进制仅展示用。 |

## 九、完整示例代码（可复制到工程）

下面给出 **`ThinkGearFrameAssembler`** 的完整示例：含 **`m_rxBuffer`**、**双 `0xAA` 对齐**、**长度 `N` 校验**、**可选整帧校验和**、**缓冲上限**。  
文件路径可按你项目习惯放在 `device/` 或新建 `protocol/`。

> **校验和规则**（与常见 ThinkGear 流一致）：对 **Payload**（帧内下标 `3` 起连续 `N` 字节）求和，取反后保留低 8 位，与最后一字节比较。若你设备文档不同，只改 `checksumOk()` 即可。

### 9.1 `thinkgearframeassembler.h`

```cpp
#ifndef THINKGEARFRAMEASSEMBLER_H
#define THINKGEARFRAMEASSEMBLER_H

#include <QByteArray>
#include <QObject>
#include <QString>

/**
 * 从任意长度的串口 chunk 中切出完整帧：
 * [0xAA][0xAA][N][Payload×N][Checksum]，总长度 4+N。
 * 不负责 TLV 解析；解析请连接 frameReady 后在另一类中处理。
 */
class ThinkGearFrameAssembler : public QObject
{
    Q_OBJECT

public:
    explicit ThinkGearFrameAssembler(QObject *parent = nullptr);

    /** 是否在 emit 前校验校验和（默认 true）。仅调试切帧可设为 false。 */
    void setVerifyChecksum(bool on) { m_verifyChecksum = on; }
    bool verifyChecksum() const { return m_verifyChecksum; }

    /** 丢弃全部积压数据（例如设备断开或换端口后调用）。 */
    void clearBuffer();

    quint64 framesEmitted() const { return m_framesEmitted; }
    quint64 checksumFailures() const { return m_checksumFailures; }
    quint64 lengthResyncs() const { return m_lengthResyncs; }
    quint64 bufferOverflows() const { return m_bufferOverflows; }

public slots:
    /** 连接 SerialPort::dataReceived 到此槽。 */
    void onBytes(const QByteArray &chunk);

signals:
    /** 一整帧原始二进制，从 0xAA 开始，长度为 4+N。 */
    void frameReady(const QByteArray &frame);

    /** 接收缓冲超过上限被清空时发出（便于打日志）。 */
    void rxBufferOverflowed(int previousSize);

private:
    void processBuffer();
    static bool checksumOk(const QByteArray &frame);

    QByteArray m_rxBuffer;
    bool m_verifyChecksum = true;

    static constexpr int kMaxRxBytes = 64 * 1024;

    quint64 m_framesEmitted = 0;
    quint64 m_checksumFailures = 0;
    quint64 m_lengthResyncs = 0;
    quint64 m_bufferOverflows = 0;
};

#endif // THINKGEARFRAMEASSEMBLER_H
```

### 9.2 `thinkgearframeassembler.cpp`

```cpp
#include "thinkgearframeassembler.h"

ThinkGearFrameAssembler::ThinkGearFrameAssembler(QObject *parent)
    : QObject(parent)
{
}

void ThinkGearFrameAssembler::clearBuffer()
{
    m_rxBuffer.clear();
}

void ThinkGearFrameAssembler::onBytes(const QByteArray &chunk)
{
    if (chunk.isEmpty())
        return;

    m_rxBuffer.append(chunk);

    if (m_rxBuffer.size() > kMaxRxBytes) {
        ++m_bufferOverflows;
        const int prev = m_rxBuffer.size();
        m_rxBuffer.clear();
        emit rxBufferOverflowed(prev);
        return;
    }

    processBuffer();
}

bool ThinkGearFrameAssembler::checksumOk(const QByteArray &frame)
{
    if (frame.size() < 4)
        return false;

    const int n = static_cast<unsigned char>(frame[2]);
    if (frame.size() != 4 + n)
        return false;

    unsigned sum = 0;
    for (int i = 0; i < n; ++i)
        sum += static_cast<unsigned char>(frame[3 + i]);

    const auto expected = static_cast<unsigned char>((~sum) & 0xFF);
    return expected == static_cast<unsigned char>(frame[3 + n]);
}

void ThinkGearFrameAssembler::processBuffer()
{
    while (true) {
        const int syncIndex = m_rxBuffer.indexOf(QByteArrayView("\xAA\xAA"));

        if (syncIndex < 0) {
            // 未找到双 AA：保留最后一个字节（可能为第一个 AA），其余丢弃
            if (m_rxBuffer.size() > 1)
                m_rxBuffer.remove(0, m_rxBuffer.size() - 1);
            return;
        }

        if (syncIndex > 0)
            m_rxBuffer.remove(0, syncIndex);

        if (m_rxBuffer.size() < 3)
            return;

        const auto n = static_cast<unsigned char>(m_rxBuffer[2]);
        if (n == 0 || n > 170) {
            ++m_lengthResyncs;
            m_rxBuffer.remove(0, 1);
            continue;
        }

        const int frameLen = 4 + static_cast<int>(n);
        if (m_rxBuffer.size() < frameLen)
            return;

        QByteArray frame = m_rxBuffer.left(frameLen);

        if (m_verifyChecksum && !checksumOk(frame)) {
            ++m_checksumFailures;
            m_rxBuffer.remove(0, 1);
            continue;
        }

        m_rxBuffer.remove(0, frameLen);
        ++m_framesEmitted;
        emit frameReady(frame);
        // 继续 while，处理粘在同一 chunk 里的下一帧
    }
}
```

### 9.4 与 `SerialPort`、解析逻辑接线（示例）

```cpp
#include "device/serialport.h"
#include "device/thinkgearframeassembler.h"

// 例如在 MainWindow 构造里：
auto *serial = new SerialPort(this);
auto *assembler = new ThinkGearFrameAssembler(this);

connect(serial, &SerialPort::dataReceived,
        assembler, &ThinkGearFrameAssembler::onBytes);

connect(assembler, &ThinkGearFrameAssembler::frameReady, this,
        [](const QByteArray &frame) {
            // TODO: 调用你的 parserData(frame, ...) 或专用解析函数
            Q_UNUSED(frame);
        });

connect(assembler, &ThinkGearFrameAssembler::rxBufferOverflowed, this,
        [](int prevSize) {
            qWarning() << "RX buffer overflow, cleared. Previous size:" << prevSize;
        });

// 打开串口
SerialPortConfig cfg;
cfg.portName = QStringLiteral("COM7");
serial->setConfig(cfg);
serial->openReadOnly();
```

### 9.5 使用说明摘要

| 项目 | 说明 |
|------|------|
| `N > 170` | 与你在解析文档中的上限一致；若官方另有规定请改 `thinkgearframeassembler.cpp` 中判断。 |
| 校验失败 | 默认丢弃并从下一字节重新对齐；`m_checksumFailures` 自增。 |
| 仅调试物理层 | `assembler->setVerifyChecksum(false)` 可只切帧不验和（仍要求 `N` 合法且长度够）。 |
| 线程 | 与 `QSerialPort` 同线程连接时，上述 `connect` 默认即可；若串口在工作线程，组帧对象也应同线程或改用 `Qt::QueuedConnection`。 |
