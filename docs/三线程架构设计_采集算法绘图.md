# 三线程架构设计（采集解析 + 算法预处理 + 日志落盘，UI仅绘图）

> 目标：按你后续需求，把流程拆成 3 个工作线程，支持：  
> 1) 串口采集 + 协议解析  
> 2) 算法缓存区预处理 + 算法推理  
> 3) CSV/TXT 落盘  
> 同时 UI 线程只做显示和用户交互，不承载重计算。

---

## 1. 线程与职责划分（最终形态）

### 线程 A：AcquisitionThread（采集解析线程）

- 持有对象：
  - `SerialPort`
  - `ThinkGearFrameAssembler`
  - `ThinkGearPayloadParser`
  - `RawtOutUvProcessor`
- 负责：
  - 串口读取、组帧、TLV 解析、`rawInt16->uv` 换算
  - 输出原始样本流（`RawPacket`）
- 不做：
  - UI 更新
  - 算法计算
  - 文件写盘

### 线程 B：AlgorithmThread（算法预处理线程）

- 持有对象：
  - `PictureAndALgBuffer`（或新建专用环形缓存）
  - 预处理器（滤波、去伪迹、归一化、分窗）
  - 算法对象（你的后续推理模块）
- 负责：
  - 接收采集线程样本，写入缓存
  - 固定窗口（如 128/256 点）预处理
  - 算法输出特征/分类/控制值
  - 产生“绘图数据块”和“算法结果块”

### 线程 C：LogThread（日志落盘线程）

- 持有对象：
  - `CsvLogWorker`
  - （可选）`TxtActionWorker`
- 负责：
  - EEG 数据写 CSV
  - 操作日志写 TXT（若你后续也想独立线程）

### UI线程（主线程）

- 持有：
  - `MainWindow` / `QCustomPlot`
- 只做：
  - 点击开始/停止
  - 接收跨线程信号并刷新图
  - 不做重计算，不直接写文件

---

## 2. 数据流（建议）

```text
SerialPort -> Assembler -> PayloadParser -> RawToUv
     (线程A) emit rawPacketReady(...)
                 |
                 v (QueuedConnection)
           AlgorithmEngine::onRawPacket(...)
                 |
      预处理 + 分窗 + 算法 (线程B)
         |                     |
         | emit plotChunkReady | emit algoResultReady
         v                     v
      UI线程绘图            UI线程显示结果

         同时：
         线程A / 线程B -> push LogItem -> LogBuffer
                                |
                                v
                        CsvLogWorker(线程C) drainBatch -> CSV
```

## 3. 建议的数据结构

```cpp
// 采集线程发给算法线程的原始样本
struct RawPacket {
    qint64 tsMs = 0;
    quint64 seq = 0;
    qint16 rawInt16 = 0;
    double rawUv = 0.0;
};

// 算法线程发给UI的绘图块
struct PlotChunk {
    QVector<double> y;     // 原始/预处理后的波形
    quint64 seqStart = 0;
    quint64 seqEnd = 0;
};

// 算法线程发给UI/日志的结果
struct AlgoResult {
    qint64 tsMs = 0;
    quint64 seqEnd = 0;
    double score = 0.0;
    QString label;
};
```

## 4. 类设计示例（按你当前工程命名风格）

## 4.1 AcquisitionEngine（线程A内对象）

### `acquisitionengine.h`

```cpp
#pragma once
#include <QObject>
#include "device/serialport.h"
#include "thinkgear/thinkgearframeassembler.h"
#include "thinkgear/thinkgearpayloadparser.h"
#include "thinkgear/rawtouvprocessor.h"

struct RawPacket {
    qint64 tsMs = 0;
    quint64 seq = 0;
    qint16 rawInt16 = 0;
    double rawUv = 0.0;
};
Q_DECLARE_METATYPE(RawPacket)

class AcquisitionEngine : public QObject {
    Q_OBJECT
public:
    explicit AcquisitionEngine(QObject *parent = nullptr);

public slots:
    void start(const QString &portName, qint32 baudRate);
    void stop();

signals:
    void rawPacketReady(const RawPacket &pkt);
    void warningMessage(const QString &msg);
    void statusMessage(const QString &msg);

private slots:
    void onRawInt16(qint16 raw);
    void onUvReady(double uv);

private:
    SerialPort *m_serial = nullptr;
    ThinkGearFrameAssembler *m_assembler = nullptr;
    ThinkGearPayloadParser *m_parser = nullptr;
    RawtOutUvProcessor *m_converter = nullptr;

    qint16 m_lastRawInt16 = 0;
};
```

### `acquisitionengine.cpp`（关键逻辑）

```cpp
#include "acquisitionengine.h"
#include <QDateTime>

AcquisitionEngine::AcquisitionEngine(QObject *parent) : QObject(parent)
{
    m_serial = new SerialPort(this);
    m_assembler = new ThinkGearFrameAssembler(this);
    m_parser = new ThinkGearPayloadParser(this);
    m_converter = new RawtOutUvProcessor(this);

    connect(m_serial, &SerialPort::dataReceived, m_assembler, &ThinkGearFrameAssembler::onBytes);
    connect(m_assembler, &ThinkGearFrameAssembler::frameReady, m_parser, &ThinkGearPayloadParser::parseFrame);
    connect(m_parser, &ThinkGearPayloadParser::rawReceived, this, &AcquisitionEngine::onRawInt16);
    connect(m_parser, &ThinkGearPayloadParser::rawReceived, m_converter, &RawtOutUvProcessor::onRaw);
    connect(m_converter, &RawtOutUvProcessor::uvValueReady, this, &AcquisitionEngine::onUvReady);
}

void AcquisitionEngine::onRawInt16(qint16 raw) { m_lastRawInt16 = raw; }

void AcquisitionEngine::onUvReady(double uv)
{
    RawPacket p;
    p.tsMs = QDateTime::currentMSecsSinceEpoch();
    p.seq = static_cast<quint64>(m_converter->sampleCount());
    p.rawInt16 = m_lastRawInt16;
    p.rawUv = uv;
    emit rawPacketReady(p);
}
```

## 4.2 AlgorithmEngine（线程B内对象）

### `algorithmengine.h`

```cpp
#pragma once
#include <QObject>
#include "core/pictureandalgbuffer.h"

struct RawPacket;
struct PlotChunk { QVector<double> y; quint64 seqStart=0; quint64 seqEnd=0; };
struct AlgoResult { qint64 tsMs=0; quint64 seqEnd=0; double score=0.0; QString label; };
Q_DECLARE_METATYPE(PlotChunk)
Q_DECLARE_METATYPE(AlgoResult)

class AlgorithmEngine : public QObject {
    Q_OBJECT
public:
    explicit AlgorithmEngine(QObject *parent=nullptr);

public slots:
    void onRawPacket(const RawPacket &pkt);   // 接收线程A样本
    void setWindowSize(int n) { m_windowSize = n; }

signals:
    void plotChunkReady(const PlotChunk &chunk); // 发给UI线程绘图
    void algoResultReady(const AlgoResult &res); // 发给UI和日志

private:
    PictureAndALgBuffer m_buf{128*80};
    int m_windowSize = 128;
};
```

### `algorithmengine.cpp`（示意）

```cpp
#include "algorithmengine.h"
#include <QDateTime>

AlgorithmEngine::AlgorithmEngine(QObject *parent) : QObject(parent) {}

void AlgorithmEngine::onRawPacket(const RawPacket &pkt)
{
    RawSample s;
    s.raw = pkt.rawUv;
    s.raw_count = pkt.seq;
    s.time = QString::number(pkt.tsMs);
    m_buf.appendRawvalue(s);

    QVector<RawSample> chunk;
    if (!m_buf.tryDequeueRawChunkForAlgo(m_windowSize, chunk))
        return;

    // 1) 预处理（示意：可替换为滤波/归一化）
    QVector<double> y;
    y.reserve(chunk.size());
    for (const auto &it : chunk)
        y.push_back(it.raw);

    // 2) 绘图块
    PlotChunk pc;
    pc.y = y;
    pc.seqStart = chunk.first().raw_count;
    pc.seqEnd = chunk.last().raw_count;
    emit plotChunkReady(pc);

    // 3) 算法结果（示意）
    AlgoResult r;
    r.tsMs = QDateTime::currentMSecsSinceEpoch();
    r.seqEnd = pc.seqEnd;
    r.score = y.isEmpty() ? 0.0 : y.last();
    r.label = QStringLiteral("demo");
    emit algoResultReady(r);
}
```

## 5. MainWindow 组装三线程（关键示例）

```cpp
// mainwindow.h
QThread *m_acqThread = nullptr;
QThread *m_algThread = nullptr;
QThread *m_logThread = nullptr;

AcquisitionEngine *m_acq = nullptr;
AlgorithmEngine *m_alg = nullptr;
CsvLogWorker *m_csvWorker = nullptr;
LogBuffer m_logBuffer{8192};
```

```cpp
// mainwindow.cpp 初始化（构造或专用 initThreads）
qRegisterMetaType<RawPacket>("RawPacket");
qRegisterMetaType<PlotChunk>("PlotChunk");
qRegisterMetaType<AlgoResult>("AlgoResult");

m_acqThread = new QThread(this);
m_algThread = new QThread(this);
m_logThread = new QThread(this);

m_acq = new AcquisitionEngine();
m_alg = new AlgorithmEngine();
m_csvWorker = new CsvLogWorker(&m_logBuffer);

m_acq->moveToThread(m_acqThread);
m_alg->moveToThread(m_algThread);
m_csvWorker->moveToThread(m_logThread);

// A -> B
connect(m_acq, &AcquisitionEngine::rawPacketReady,
        m_alg, &AlgorithmEngine::onRawPacket, Qt::QueuedConnection);

// B -> UI
connect(m_alg, &AlgorithmEngine::plotChunkReady, this, [this](const PlotChunk &pc){
    // 在UI线程更新QCustomPlot
    Q_UNUSED(pc);
});
connect(m_alg, &AlgorithmEngine::algoResultReady, this, [this](const AlgoResult &res){
    // 在UI线程显示算法结果
    Q_UNUSED(res);
});

// A/B -> LogBuffer（示例：在各自回调里 push）
connect(m_acq, &AcquisitionEngine::rawPacketReady, this, [this](const RawPacket &p){
    LogItem li;
    li.tsMs = p.tsMs;
    li.seq = p.seq;
    li.rawInt16 = p.rawInt16;
    li.rawUv = p.rawUv;
    m_logBuffer.push(li);
});

// 启动线程后再启动worker
connect(m_logThread, &QThread::started, m_csvWorker, &CsvLogWorker::start);

m_acqThread->start();
m_algThread->start();
m_logThread->start();
```

## 6. 启停顺序（非常关键）

### Start

1. 配置 CSV 路径 -> `m_csvWorker->setOutputPath(...)`
2. 启动三个线程（若未启动）
3. 通过 `invokeMethod` 在线程A调用 `AcquisitionEngine::start(port, baud)`

### Stop

1. 在线程A调用 `AcquisitionEngine::stop`
2. 在线程C调用 `CsvLogWorker::stop`
3. `quit + wait` 结束三个线程

## 7. 迁移建议（从你现在代码渐进）

1. **先复用你已有** `ThinkGearLinkTester`，把采集链路拆进 `AcquisitionEngine`（线程A）。  
2. `PictureAndALgBuffer` 与预处理/算法逻辑搬到 `AlgorithmEngine`（线程B）。  
3. `CsvLogWorker` 保持线程C。  
4. `MainWindow` 只保留 UI 操作与信号接收。

## 8. 你当前代码里优先要修的一行

在 `ThinkGearLinkTester::onRawUvReady` 里：

```cpp
li.tsMs = now.toString(Qt::TextDate); // 错误
```

应改为：

```cpp
li.tsMs = QDateTime::currentMSecsSinceEpoch();
```

否则 CSV 的时间戳列语义会错。

## 9. 验收标准

- UI流畅：开始后界面不卡顿  
- `logQ` 不长期上涨，CSV持续写入  
- 算法线程可稳定产出 `plotChunk` 与 `algoResult`  
- 停止后线程无残留、无跨线程 timer 报错
